## 📁 生成的文件说明

### 📄 员工信息数据库.db

**类型：** SQLite 数据库文件

**查看方式：**
- 方式1：使用 SQLite 客户端（推荐）
  ```bash
  # 安装 SQLite 客户端
  # macOS: brew install sqlite3
  # Windows: 下载 SQLite Studio
  
  # 打开数据库
  sqlite3 员工信息数据库.db
  
  # 查看表
  .tables
  
  # 查询数据
  SELECT * FROM staff;
  
  # 退出
  .quit
  ```

- 方式2：使用 VS Code 扩展
  - 安装 "SQLite" 扩展
  - 右键点击 .db 文件 → "Open Database"

- 方式3：使用 Python 查询
  ```python
  import sqlite3
  conn = sqlite3.connect("员工信息数据库.db")
  cursor = conn.cursor()
  cursor.execute("SELECT * FROM staff;")
  print(cursor.fetchall())
  ```

**文件内容：**
- 表名：`staff`
- 记录数：4条（删除了赵六后）
- 字段：id, name, department, salary, age

## 💬 面试讲解要点

### 核心问题1️⃣：项目做了什么？

**答案框架：**
```
这个项目演示了一个完整的员工信息管理系统，主要展示了：

1. 数据库设计与创建
   - 设计了包含 id, name, department, salary, age 的表结构
   - 使用主键和自增ID确保数据完整性

2. 数据操作（CRUD）
   - 创建（Create）：INSERT INTO 插入员工信息
   - 读取（Read）：SELECT 进行各种查询
   - 修改（Update）：UPDATE 更新李四的薪资
   - 删除（Delete）：DELETE 删除行政部员工

3. 查询技巧
   - 条件查询：WHERE 部门过滤
   - 范围查询：WHERE salary > 6000
   - 分组统计：GROUP BY 部门统计

4. 数据导出
   - 用 pandas 读取数据库
   - 导出为 Excel 文件便于分析
```

---

### 核心问题2️⃣：技术难点有哪些？

**答案框架：**
```
主要技术点包括：

1. Python sqlite3 模块的使用
   - 数据库连接管理
   - SQL 语句执行
   - 事务提交（commit）保证数据一致性

2. SQL 语言的应用
   - DDL（数据定义）：CREATE TABLE
   - DML（数据操作）：INSERT, SELECT, UPDATE, DELETE
   - 条件筛选和范围查询
   - 分组聚合（GROUP BY, COUNT, AVG）

3. 数据持久化
   - 确保 commit() 后数据写入磁盘
   - 防止未提交的事务丢失

4. 数据导出集成
   - pandas 读取数据库数据
   - 转换为 DataFrame 格式
   - 导出为 Excel 文件
```
